Angular Pipes

Transform displayed values in templates without changing the underlying data.

What Are Pipes?

  • Formatting: Pipes format values using | syntax (e.g. date, currency, percent).
  • Async: The async pipe subscribes to Observables, renders the latest value, and unsubscribes automatically.
  • Presentation-only: Pipes change how a value is displayed, not the underlying data.
  • Pure by default: Pure pipes run when input references change; avoid in-place mutation of arrays/objects.

Basic Pipes

Format strings, numbers, dates, and more with built-in pipes. Many accept options (e.g. currency:'USD', date:'short').

{{ title | uppercase }} uppercase
{{ price | currency:'USD' }} currency:'USD'
{{ today | date:'mediumDate' }} date:'mediumDate'
{{ percent | percent:'1.0-2' }} percent:'1.0-2'

Async Pipe

Render the latest value from an Observable. Unsubscribes automatically when the view is destroyed.

{{ time$ | async | date:'mediumTime' }} async → date:'mediumTime'

Custom Pipe

Build small, reusable formatters with @Pipe. Mark as standalone and import into the component. Pure by default; avoid impure pipes unless necessary.

{{ text }} original
{{ text | titlecase2 }} titlecase2 (custom)

Pipe Syntax

Pipes use the | operator inside template expressions. They accept zero or more colon-separated arguments and can be chained left to right.

Built-in Pipe Reference

Angular ships with a set of built-in pipes organised by category. All are standalone and available via CommonModule.

String Pipes

Pipe Description
uppercase Converts string to UPPER CASE
lowercase Converts string to lower case
titlecase Capitalises the first letter of each word

Number Pipes

Pipe Description
number Formats a number as a decimal string (e.g. 1,234,567.89)
percent Multiplies by 100 and appends %
currency Formats as currency with symbol (e.g. $1,234.50)

Other Pipes

Pipe Description
json Serialises an object to a JSON string (useful for debugging)
slice Returns a shallow copy of a portion of an array or string
async Subscribes to an Observable/Promise and returns the latest value

Json Pipe — Debugging

The json pipe serialises any value into a readable JSON string. Useful for quick debugging in templates.

Chaining Pipes

Combine multiple pipes by chaining them. Evaluation flows left to right: the output of one pipe becomes the input of the next.

{{ chainText }} original
{{ chainText | uppercase }} uppercase
{{ chainText | titlecase }} titlecase
{{ chainText | uppercase | slice:0:15 }} uppercase → slice:0:15
{{ chainText | lowercase | titlecase }} lowercase → titlecase
{{ (numValue | currency:'USD':'symbol':'1.2-2') }} number → currency

Creating a Custom Pipe

Use the @Pipe decorator to create your own. The transform method receives the input value followed by any arguments. Mark as standalone: true and import it into any component that needs it.
