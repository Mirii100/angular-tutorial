Angular Fundamentals

Click any lecture below to expand its syntax, semantics, and live examples.

Syntax

Semantics

Double curly braces evaluate a TypeScript expression and convert the result to a string. Angular watches the component for changes and re-renders the text automatically. You can use any valid JS expression: arithmetic, ternaries, method calls, and property access.

Live Example

{{ points[3] }}

{{ points[2] }}

{{ (ratio || 0).toFixed(2) }}

Syntax

Semantics

Square brackets [property] tell Angular to evaluate the expression on the right and assign the result to the DOM property on the left. This is one-way binding from component to view. The binding updates whenever the component state changes.

Live Example

You wrote: {{ firstName }}

Syntax

Semantics

Parentheses (event) bind a DOM event to a component method. Angular listens for the native event and calls your handler. The $event object is the native DOM event. Modifiers like .enter, .blur, or .debounce(400) filter or transform when the handler fires.

Live Example

Count: {{ count }}

Value: {{ value }}

Last key: {{ lastKey }}

Syntax

Semantics

Two-way binding combines property binding [value] with event binding (input) to keep the model and view in sync. When the user types, the event handler updates the component property, which re-renders the view. Angular also offers [(ngModel)] with FormsModule for shorthand.

Live Example

Total: ${{ (quantity || 0) * (price || 0) }}

Hello {{ name || 'friend' }}!

Favorite: {{ favorite }}

Syntax

Semantics

*ngIf adds or removes an element from the DOM based on a truthy/falsy expression. When false, the element is completely removed (not hidden). The else clause references an ng-template to render when false. The then; else syntax lets you specify both templates explicitly.

Syntax

Semantics

*ngFor iterates over an array and stamps out a template per item. Angular tracks each item by identity. When the array reference changes, it diffs and updates the DOM. trackBy provides a stable key (like item.id) so Angular reuses existing DOM nodes instead of recreating them, preserving focus, animations, and input state.

Syntax

Semantics

Use [attr.name] when you need to bind to an HTML attribute (not a DOM property). This is essential for attributes like colspan, rowspan, aria-label, and title that don't have corresponding DOM properties. The attribute value is set as a string, and Angular updates it when the expression changes.

Live Example

{{ span }}
A B C
colspan={{ span }} B C
A B C
Row 1 Cell Cell

Syntax

Semantics

Pipes transform displayed values without changing the underlying data. The pipe operator | takes the left expression and passes it through the pipe function. You can chain pipes and pass arguments with colons. Common built-in pipes: date, currency, number, uppercase, lowercase, json, slice.

Live Example

{{ today | date:'yyyy-MM-dd' }}

Syntax

Semantics

An attribute directive attaches to an existing element and augments its behavior. @HostListener listens to DOM events on the host element. @HostBinding binds a host property (like style) to a class field. The directive is declared with a selector attribute name and imported where needed. Standalone directives don't require an NgModule.

Syntax

Semantics

@Input() creates a bindable property that the parent sets via property binding. @Output() creates an event emitter; the parent listens with parentheses. Data flows down via inputs, events flow up via outputs. The EventEmitter extends RxJS Subject and calls emit() to broadcast values.

Live Example

Last child emitted: {{ lastCount }}

Syntax

Semantics

Debouncing delays the update until the user pauses typing, reducing unnecessary processing. Manual debounce uses clearTimeout/setTimeout. Angular also supports (input.debounce) in templates. Event modifiers like .enter, .escape, .tab filter which key triggers the handler.
